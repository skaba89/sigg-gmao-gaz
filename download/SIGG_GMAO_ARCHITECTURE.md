| Principe | Description |
|----------|-------------|
| **Separation des responsabilites** | Chaque couche a un role clairement defini (presentation, logique metier, donnees) |
| **API-First** | Toutes les fonctionnalites sont accessibles via l'API REST |
| **Stateless** | Les serveurs d'application ne maintiennent pas d'etat session |
| **Defense en profondeur** | Securite multi-couches (WAF, RBAC, JWT, chiffrement) |
| **Observabilite** | Logs structures, metriques, tracing distribue |
| **Resilience** | Redondance, circuit breakers, graceful degradation |

### 8.2 Flux de Donnees IoT

```
Capteur вҶ’ Edge Gateway вҶ’ MQTT Broker вҶ’ IoT Gateway вҶ’ Traitement вҶ’ Actions

1. [Capteur] Lit valeur (ex: pression = 8.5 bar)
2. [Edge Gateway] Agrege + timestamp вҶ’ MQTT publish: sigg/sensors/pressure
3. [MQTT Broker] Route le message aux subscribers
4. [IoT Gateway] Recoit et valide les donnees
5. [Threshold Check] Compare avec les seuils de EquipmentParameter
6. Si depassement:
   вҶ’ [IncidentService] Cree un incident automatique
   вҶ’ [NotificationService] Alerte les responsables
   вҶ’ [EquipmentService] Met a jour le health score
7. [Predictive Analytics] Analyse les tendances
8. [MaintenanceService] Genere un plan preventif si necessaire
```
